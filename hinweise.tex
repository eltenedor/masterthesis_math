\section{M\"oglichkeiten Mathematik mit \LaTeX{} zu setzen}\label{section:mathematikmodi}
Der Zweck dieses Abschnitts ist das Erinnern an die wichtigsten Mathematikmodi in \LaTeX.

\subsection{Inline Math Modus}\label{subsection:inlinemath}\index{Inline Math Modus}
Will man Formeln und mathematische Symbole fließend innerhalb eines Textes setzen, so kann man dafür den sogenannten \glqq{}Inline Math Modus\grqq{} benutzen. Dieser kann auf folgende Arten aufgerufen werden, wobei es noch mehr Möglichkeiten gibt:
\begin{enumerate}[1.]
	\item \begin{verbatim}$ Formel $\end{verbatim}
  \item \begin{verbatim}\( Formel \)\end{verbatim}
\end{enumerate}
 
\begin{Beispiel}
\begin{itemize}\item Formel und ihr \LaTeX{}-Code: Es gilt $a^2+b^2=c^2$. \begin{verbatim}Es gilt $a^2+b^2=c^2$.\end{verbatim}
   \item Weitere Formel und ihr \LaTeX{}-Code: Es gilt \(g^ng^m=g^{m+n}\).
   \begin{verbatim}Es gilt \(g^ng^m=g^{m+n}\).\end{verbatim}
  \end{itemize}
\end{Beispiel}

\subsection{Display Math Modus}\label{subsection:displaymath}\index{Display Math Modus}
Ist man daran interessiert Formeln oder Gleichungen besonders hervorzuheben, zum Beispiel in dem diese in einer eigenen Zeile stehen, so sollte man den \glqq{}Display Math Modus\grqq{} benutzen. Außerdem hat man hier auch die Möglichkeit, Gleichungen zu markieren, damit man später auf diese verweisen kann. Dazu später mehr in Abschnitt \ref{section:verweise}.

Es gibt folgende M\"oglichkeiten, diesen Modus zu benutzen:

\begin{enumerate}[1.]
	\item Eckige Klammern: Setzt die Formel in eine eigene Zeile und man hat die M\"oglichkeit, durch den Befehl \verb|\tag{}| die Formel mit einem Symbol zu versehen.\index{Display Math Modus!Eckige Klammern}
	\begin{verbatim}\[ Formel \tag{optional}\]\end{verbatim}

	\item equation-Umgebung\index{Display Math Modus!equation-Umgebung}: Setzt die Formel in eine  eigene Zeile und die Formel wird automatisch nummeriert.
	\begin{verbatim}
	\begin{equation}
	Formel
	\end{equation}
	\end{verbatim}
	
	\item equation*-Umgebung: Setzt die Formel in eine eigene Zeile, ohne die Formel zu nummerieren.
	\begin{verbatim}
	\begin{equation*}
	Formel
	\end{equation*}
	\end{verbatim}

	\item align-Umgebung\index{Display Math Modus!align-Umgebung}: Erlaubt es, Formeln \"uber mehrere Zeilen ausgerichtet darzustellen. Dabei wird jede Zeile nummeriert. Zeilenumbr\"uche werden mit \verb|\\| durchgef\"uhrt und die Formeln werden so ausgerichtet, dass die direkt auf \verb|&| folgenden Zeichen untereinander stehen. M\"ochte man, dass eine Zeile nicht nummeriert wird, so schreibt man am Ende der Zeile \verb|\nonumber|

	\begin{verbatim}
	\begin{align}
	 Formel1 &= Formel2\\
		 &= Formel3
	\end{align}
	\end{verbatim}

	\item align*-Umgebung: Funktioniert genauso wie die align-Umgebung. Es werden jedoch gar keine Zeilen nummeriert.
	
	\begin{verbatim}
	\begin{align*}
	 Formel1 &= Formel2\\
		 &= Formel3
	\end{align*}
	\end{verbatim}
\end{enumerate}

\section{Definitionen, Lemmata, Sätze etc.}\label{section:theoremumgebung}
Mathematische Texte werden üblicherweise durch spezielle Absätze wie Definitionen, Lemmata, Propositionen, Sätze, Korollare, Beweise, etc. strukturiert. In \LaTeX{} gibt es entsprechende Umgebungen, die das leisten.

\subsection{\AmS-\LaTeX{} und theorem-Umgebungen}\label{subsection:amsthm}Das Paket \texttt{amsthm} in \AmS-\LaTeX{} liefert ein mächtiges Werkzeug zum Anlegen der eben erw\"ahnten Umgebungen. Es handelt sich um sogenannte \emph{theorem-Umgebungen}\index{theorem-Umgebung}. Wir erl\"autern kurz deren Struktur:
\begin{enumerate}
	\item Im Kopfteil des \LaTeX-Dokumentes muss zunächst das \emph{Paket} \texttt{amsthm} geladen werden, das in jeder bekannten 
\LaTeX-Distribution installiert ist. In dieser Vorlage wird das Paket in der eingebundenen Datei \texttt{pakete.tex} geladen.

	\item Anschließend sucht man sich einen der drei Stile aus (unterscheidet Formatierung für Aussagen, Definitionen oder Bemerkungen) und ruft diesen im Kopfteil auf. In dieser Vorlage geschieht dieser Teil in der Datei \texttt{befehle.tex}.
	
	\item Schließlich kann man direkt danach eine Umgebung definieren. Wie das genau funktioniert, sehen wir gleich.
\end{enumerate}

\subsection{Eine theorem-Umgebung definieren}\label{subsection:defthmumgebung}

In \texttt{amsthm} werden folgende \emph{Stile}\index{theorem-Umgebung!Stil definieren} unterschieden:
	\begin{itemize}
		\item Durch Verwendung von \verb|\theoremstyle{plain}| im Kopfteil wird der Stil für \emph{Aussagen} geladen (Satz, Korollar, Lemma, Proposition).
		\item Mit \verb|\theoremstyle{definition}| wird der Stil für \emph{Definition} aufgerufen. Auch \emph{Beispiele} können hier untergeordnet werden.
		\item Mit dem Befehl \verb|\theoremstyle{remark}| werden \emph{Bemerkungen} aufgerufen. Auch \emph{Beispiele} kann man diesem Stil unterordnen.
	\end{itemize}
Man kann auch eigene Stile definieren, aber benutzen Sie bitte f\"ur die Standardumgebungen auch die vordefinierten Stile.
	
\subsubsection{Anleitung zur Definition einer theorem-Umgebung}\label{subsubsection:theoremanleitung}
\begin{enumerate}
	\item Legen Sie im Kopfteil den Stil fest durch \verb|\theoremstyle{Stil}|.
	\item Durch \verb|\newtheorem{Umgebung1}{Name1}[Option]| definiert man eine \emph{nummerierte} theorem-Umgebung. 
	
	Dabei kann man optional entscheiden, ob der Z\"ahler f\"ur Umgebung1 -- zum Beispiel nach jedem Aufrufen von \verb|\chapter{}| oder \verb|\section{}| -- auf $0$ gesetzt wird. Dafür trägt man als Option \texttt{chapter} oder \texttt{section} ein.
	\item Legt man eine weitere theorem-Umgebung an, so kann die Nummerierung vererbt werden: \verb|\newtheorem{Umgebung2}[Umgebung1]{Name2}|
	\item Mit \verb|\newtheorem*{Umgebung3}{Name3}| wird keine Nummerierung erzeugt.
\end{enumerate}

\subsection{Kein Satz ohne Beweis: Die proof-Umgebung}\label{subsection:proofumgebung}
In \texttt{amsthm} ist bereits eine proof-Umgebung\index{proof-Umgebung} vordefiniert, welche -- abhängig von der verwendeten Sprache -- durch \glqq{}Beweis.\grqq{} oder \glqq{}Proof.\grqq{} eingeleitet wird und auf $\square$ endet.

\begin{verbatim}
\begin{proof}

\end{proof}
\end{verbatim}


In manchen Fällen möchte man gerne statt \glqq{}Beweis.\grqq{} eine andere Bezeichnung benutzen. Dies geht wie folgt:

\begin{verbatim}
\begin{proof}[Eigener Name]

\end{proof}
\end{verbatim}

Endet ein Beweis auf eine im Display Math Modus gesetzte Formel, so wird das QED-Symbol $\square$ eine Zeile unter der Formel gesetzt, was nicht schön aussieht:

\begin{proof}Es folgt
\[a^2=b^2+c^2\mbox{.}\]
\end{proof}

\begin{verbatim}
\begin{proof}Es folgt
\[a^2=b^2+c^2\mbox{.}\]
\end{proof}
\end{verbatim}

Der Befehl \verb|\qedhere| ist eine Lösung für dieses Problem:

\begin{proof}Es folgt
\[a^2=b^2+c^2\mbox{.}\qedhere\]
\end{proof}

\begin{verbatim}
\begin{proof}Es folgt
\[a^2=b^2+c^2\mbox{.}\qedhere\]
\end{proof}
\end{verbatim}

\subsection{Beispiel für den Einsatz von theorem-Umgebungen}\label{subsection:beispielthm}\index{theorem-Umgebung!Beispiel für}
Folgendes Beispiel illustriert den Einsatz der theorem-Umgebungen. Den Quelltext können Sie sich in der Datei \texttt{hinweise.tex} anschauen.

Der Bairesche Kategoriensatz aus der Topologie ist das grundlegende Hilfsmittel dieses Kapitels und darüberhinaus hat er noch interessante Anwendungen, wie zum Beispiel eine Aussage über stetige und nirgends differenzierbare Funktionen:
\begin{Satz}[Bairescher Kategoriensatz]\label{satzbaire}Es sei $(X,d)$ ein vollständiger metrischer Raum. Für alle $n\in\N$ sei $O_n$ offen und dicht in $X$. Dann ist $O:=\bigcap_{n\in\N}O_n$ dicht in $X$.\end{Satz}

\begin{Bemerkung}Die Aussage des Satzes bleibt bestehen, wenn man den vollständigen metrischen Raum $(X,d)$ durch einen kompakten Hausdorff-Raum $(X,\mathcal{T})$ ersetzt. Siehe dazu \cite[Theorem 48.2]{Munkres}.\end{Bemerkung}

Wir illustrieren den Satz an einem Beispiel bevor wir ihn beweisen:
\begin{Beispiel}Sei $A\subseteq\R$ abzählbar, d.h. $A=\left\{a_n\colon n\in\N\right\}$ für eine Folge $(a_n)_{n\in\N}$ in $\R$. Dann ist $O_n:=\R\setminus\{a_n\}$ dicht in $\R$ und somit ist $A^c=\bigcap_{n\in\N}O_n$ dicht in $\R$. Im Fall $A=\Q$ sieht man hiermit ein, dass die Menge der irrationalen Zahlen dicht in $\R$ liegt.\end{Beispiel}

Folgende Variante des Satzes werden wir benötigen:

\begin{Korollar}\label{korbaire}Sei $(X,d)$ ein vollständiger metrischer Raum und $A_n$ abgeschlossen sowie $A:=\bigcup_{n\in\N}A_n$. Enthält $A$ einen inneren Punkt, dann existiert $n_0$, so dass $A_{n_0}$ einen inneren Punkt enthält.\end{Korollar}

\begin{proof}Angenommen für alle $n\in\N$ besäße $A_n$ keinen inneren Punkt. Dann wäre für alle $n\in\N$ die Menge $O_n:=A_n^c$ offen und dicht in $X$. Dann müsste aber auch \[O:=\bigcap_{n\in\N}O_n=\bigcap_{n\in\N}A_n^c=\left(\bigcup_{n\in\N} A_n\right)^c=A^c\]
dicht in $X$ liegen, im Widerspruch dazu, dass $A$ einen inneren Punkt enthält.\end{proof}

\begin{proof}[Beweis des Satzes \ref{satzbaire}]In diesem Beispiel lassen wir den Beweis lieber aus. Man findet einen Beweis in \cite[Theorem 48.2]{Munkres}.\end{proof}

\section{Abbildungen}\label{section:abbildungen}
Zum Einbinden von Abbildungen sollten Sie die figure-Umgebung\index{figure-Umgebung} benutzen:

\begin{verbatim}
\begin{figure}
\includegraphics[width=\columnwidth]{filename}
\caption{Bildunterschrift}
\label{Verweismarke}
\end{figure}
\end{verbatim}

Dabei können Sie mit \verb|\includegraphics| folgende Dateitypen einbinden:
\begin{itemize}
	\item Verwenden Sie \texttt{latex} zum Kompilieren, so können Sie EPS-Dateien einbinden.
	
	\item Verwenden Sie \texttt{pdflatex} zu Kompilieren, so können Sie die Grafiktypen PDF, JPG und PNG einbinden.
\end{itemize}

Als Beispiel binden wir hier das Logo der TU Darmstadt ein.

\end{figure}

\section{Literaturverzeichnis anlegen}\label{section:literatur}\index{Literaturverzeichnis}

Es gibt im Wesentlichen zwei Möglichkeiten, ein Literaturverzeichnis anzulegen:
\begin{enumerate}
	\item Die Literaturquellen können direkt in den Text eingefügt werden. Dieser Weg ist aufwendiger, vor allem wenn man häufig die gleiche Literatur zitiert, und wird deshalb hier \emph{nicht} vorgestellt.

	\item Die Benutzung von Bib\TeX{} erlaubt eine systematischere und flexiblere Lösung, und diese M\"oglichkeit werden wir in diesem Abschnitt erl\"autern.
\end{enumerate}

\subsection{Grundlegendes zu Bib\TeX}\label{subsection:bibtex}\index{Bib\TeX}

Grundlegend m\"ussen Sie bei der Benutzung von Bib\TeX{} wie folgt vorgehen:

\begin{enumerate}
	\item Erstellen Sie eine \texttt{BIB}-Datei im Verzeichnis des \LaTeX-Quelltextes. In dieser Datei werden alle Quellen eingetragen. Je nach Quellentyp gibt es verschiedene Pflichtfelder und optionale Felder, aber dazu mehr in Abschnitt \ref{subsection:quellen}. Bearbeiten Sie diese Datei am besten in dem benutzten \LaTeX-Editor.

	\item Legen Sie im \LaTeX-Quelltext den \emph{Stil} für das Literaturverzeichnis fest: Dies geschieht durch das Laden einer \texttt{BST}-Datei.

	In der Vorlage erfolgt dies in der Datei \texttt{literaturverzeichnis.tex} via
	
	 \verb|\bibliographystyle{Stil}|.

	\item Gehört die \texttt{BST}-Datei nicht zur \LaTeX-Distribution, so m\"ussen Sie diese im Verzeichnis des \LaTeX-Quelltextes abspeichern (oder als Stil in der \LaTeX-Distribution installieren).

	In dieser Vorlage benutzen wir den Stil \texttt{alphadin}.

        \item Binden Sie die \texttt{BIB}-Datei in den \LaTeX-Quelltext ein.

	In dieser Vorlage geschieht dies in der Datei \texttt{literaturverzeichnis.tex} durch \verb|\bibliography{thesis}|.

	\item Kompilieren Sie die \texttt{BIB}-Datei mit Bib\TeX{}. Bei den meisten Editoren wird Bib\TeX{} automatisch aufgerufen, wenn Sie den \LaTeX-Quelltext kompilieren.

	\item Durch das Kompilieren mit Bib\TeX{} wird eine \texttt{BBL}-Datei erstellt. Es ist wichtig, dass diese \texttt{BBL}-Datei den gleichen Namen tr\"agt wie das zu kompilierende \LaTeX-Dokument.	
\end{enumerate}

\subsection{Quellen in BIB-Datei eintragen}\label{subsection:quellen}\index{Bib\TeX!Quellen eintragen}

Öffnen Sie zum Beispiel TeXnicCenter und wählen Sie beim Speichern das Dateiformat Bib\TeX{} aus. Darin schreiben Sie solche Eintr\"age:
\begin{verbatim}
@TYP{label,
feld1 = {feld1inhalt},
.
.
feldn = {feldninhalt}
}
\end{verbatim}

Beachten Sie folgende Hinweise zu den Eintr\"agen:

\begin{enumerate}
        \item Das Label muss eindeutig sein.
	\item Alle Felder müssen durch Kommata getrennt werden.
	\item Eine Übersicht aller Typen mit Angabe von erforderlichen und optionalen Feldern ist hier zu finden:
	\texttt{http://de.wikipedia.org/wiki/BibTeX} (unter Punkt 3, den Literaturtypen)
\end{enumerate}

\subsection{Beispiel f\"ur Literaturverzeichnis}\label{subsection:beispielliteratur}\index{Literaturverzeichnis!Beispiel}
In dieser Vorlage haben wir auch ein Literaturverzeichnis angelegt, welches Sie sich als Beispiel f\"ur die Benutzung von Bib\TeX{} anschauen sollten:

\begin{itemize}
 \item In der Datei \texttt{literaturverzeichnis.tex} wird der Stil festgelegt und die \texttt{BIB}-Datei \texttt{thesis.bib} geladen.

 \item In \texttt{thesis.bib} sehen Sie, wie Quellen eingetragen werden k\"onnen.
\end{itemize}

\subsection{Hinweis: Zitationsweisen}\label{subsection:zitation}
Reicht man einen Artikel bei einer Fachzeitschrift ein, so wird von dieser penibel vorgeschrieben, wie
\begin{itemize}
	\item das Literaturverzeichnis zu gestalten und zu ordnen ist,
	\item auf welche Weise Literaturverweise im Text stattfinden.
\end{itemize}
(Leider bei jeder Zeitschrift anders \ldots)

Für Abschlussarbeiten gibt es prinzipiell mehr Freiheiten, wobei üblicherweise eines der beiden folgenden Zitationssysteme verwendet wird:
\begin{itemize}

	\item Referenznummernsystem.
\end{itemize}
Sie sollten daher Ihren Betreuer nach dem bevorzugten Zitationssystem fragen, welches Sie in Ihrer Abschlussarbeit benutzen sollen.

\section{Titelseite und weitere Verzeichnisse}\label{section:section}
In diesem Abschnitt gehen wir nicht auf die Details ein, sondern erl\"autern nur, wie Sie in dieser Vorlage \"Anderungen vornehmen k\"onnen.

\subsection{Titelseite}\label{subsection:titel}

Es gibt im Grunde zwei Arten, in \LaTeX{} die Titelseite\index{Titelseite} zu gestalten:

\begin{enumerate}
	\item Eine Klasse hat schon eine vorgefertigte Titelseite, nur noch die Inhalte (Titel, Autor, Datum etc.) müssen eingefügt werden.
	\item Man gestaltet die Titelseite individuell.
\end{enumerate}

In dieser Vorlage wurde die Titelseite individuell gestaltet und Sie k\"onnen sich den Quelltext dazu in der Datei \texttt{titel.tex} anschauen. Sie k\"onnen an den entsprechenden Stellen Ihre Daten eintragen (Typ der Arbeit, Titel der Arbeit, Name, Datum der Abgabe, Name des Betreuers und Name des zweiten Gutachters).

\subsection{Inhaltsverzeichnis}\label{subsection:inhalt}

Das Inhaltsverzeichnis\index{Inhaltsverzeichnis} funktioniert prinzipiell wie folgt:
\begin{itemize}
	\item Alle automatisch nummerierten Überschriftarten werden zum Inhaltsverzeichnis hinzugefügt.
	\item Man kann entscheiden dieses Inhaltsverzeichnis anzuzeigen oder nicht.
	\item Man kann die Darstellungstiefe, d.h. bis zu welchem Hierarchielevel Überschriften hinzugefügt werden, auch einstellen.
\end{itemize}

In dieser Vorlage wird die Klasse \texttt{report} benutzt, so dass folgende \"Uberschriftarten automatisch nummeriert werden und im Inhaltsverzeichnis angezeigt werden:
\begin{itemize}
 \item \verb|\chapter{Kapitel}|,
 \item \verb|\section{Abschnitt}|,
 \item \verb|\subsection{Unterabschnitt}|.
\end{itemize}

Der \"Uberschriftentyp \verb|\subsubsection{Unter-Unterabschnitt}| wird nicht automatisch nummeriert und im Inhaltsverzeichnis angezeigt, kann aber dazu genutzt werden den Text besser zu gliedern.

\subsection{Stichwortverzeichnis}\label{subsection:stichwort}\index{Stichwortverzeichnis}

In jeder Abschlussarbeit sollte auch ein Stichwortverzeichnis enthalten sein. Dieses enth\"alt alphabetisch sortierte Stichw\"orter mit zugeh\"origer Seitenzahl und erm\"oglichst somit ein gezieltes Nachschlagen einzelner Begriffe.

Bereits beim Verfassen Ihrer Arbeit sollten Sie die Begriffswahl planen. Sie k\"onnen nach folgenden Fragen vorgehen:

\begin{itemize}
 \item Ist der Begriff von zentraler Bedeutung?
 
 \item Wurde der Begriff ausf\"uhrlich behandelt?
\end{itemize}

\subsubsection{Einbindung mit \LaTeX}\index{Stichwortverzeichnis!Einbindung mit \LaTeX}

In \LaTeX{} gibt es das Paket \texttt{makeidx}. In dieser Vorlage wird \texttt{makeidx} in der Datei \texttt{pakete.tex} geladen, und die Anweisung \verb|\makeindex| gegeben, um die Indexeintr\"age zu sammeln.

\subsubsection{Eintr\"age im Stichwortverzeichnis erstellen}\index{Stichwortverzeichnis!Erstellung von Eintr\"agen}

Der Befehl \verb|\index{Schluesselwort}| erzeugt einen Eintrag im Stichwortverzeichnis mit der Bezeichnung \glqq{}Schluesselwort\grqq{} und verweist auf die aktuelle Seite. Beachten Sie:

\begin{itemize}
 \item Der Befehl wird in der Regel hinter das Wort geschrieben, welches im Stichwortverzeichnis erscheinen soll.

 \item Benutzen Sie auf der gleichen Seite mehrmals den gleichen Befehl, so wird es nur einen Eintrag geben.

 \item Benutzen Sie auf verschiedenen Seiten mehrmals den gleichen Befehl, so werden alle zugeh\"origen Seitenzahlen im Stichwortzeichnis eingetragen.

 \item Man kann auch Untereintr\"age benutzen um das Stichwortverzeichnis zu strukturieren:

\verb|\index{Haupteintrag!Untereintrag!Unter-Untereintrag}|
\end{itemize}

\subsection{Sonstige Verzeichnisse}\label{subsection:verzeichnisse}

Manchmal ist es auch n\"utzlich, separate Verzeichnisse f\"ur Abbildungen, Tabellen oder auch f\"ur die verwendete Notation zu benutzen. Sie sollten mit Ihrem Betreuer kl\"aren, ob dies erw\"unscht ist.

\section{Verweise}\label{section:verweise}
In \LaTeX{} werden Sätze etc. automatisch nummeriert (man kann die Nummerierung auch unterdrücken). Gleiches gilt für Kapitel usw. Beim Verweisen kann man sich dies zu Nutze machen. Dabei sollte man die folgenden Punkte beachten:\index{Verweise}\index{Verweismarke}

\begin{itemize}
	\item \verb|\label{Verweismarke}| definiert eine Verweismarke. Der Übersicht halber definiert man die Verweismarke am besten innerhalb einer Gleichung im Display Math Modus, innerhalb einer theorem-Umgebung oder direkt nach dem chapter-Befehl etc.
	
	\item Es gibt verschiedene Möglichkeiten, sich auf eine Verweismarke zu beziehen:
	\begin{enumerate}[1.]
			\item \verb|\ref{Verweismarke}| setzt die Nummer des markierten Objektes.
			\item \verb|\eqref{Verweismarke}| setzt die Gleichungsnummer in runde Klammern.
			\item \verb|\pageref{Verweismarke}| setzt die Nummer der Seite, auf welcher die Verweismarke definiert wurde.
	\end{enumerate}
	
	\item Mit \verb|\cite[Option]{Verweismarke}| verweist man auf Literatur.
\end{itemize}
\section{Aufschreiben von Mathematik}\label{section:aufschreiben}
Es gibt ein paar Fein- und Besonderheiten, die man beim Aufschreiben von Mathematik beachten sollte. 

\subsection{Regeln}\label{subsection:regeln}
Wir haben hier -- auch unter Benutzung von \cite{Halmos} und \cite{Knuth} -- einige \glqq{}Regeln\grqq{} zur mathematischen Prosa zusammengefasst.\index{Regeln zum Aufschreiben}

\begin{Regel}
Fangen Sie keinen (Neben-)Satz mit einem Symbol an.
\end{Regel}

\begin{RBsp}\begin{itemize}
	\item Böse: $x^n-a$ hat $n$ komplexe Nullstellen.

	\item Gut: Das Polynom $x^n-a$ hat $n$ komplexe Nullstellen.
\end{itemize}
\end{RBsp}

\begin{Regel}Verwenden Sie verbale Formulierungen anstelle von vielen aufeinander folgenden logischen Symbolen, zum Beispiel $\implies$, $\forall$, 
$\exists$ oder $\ni$, sofern es sich nicht um eine Arbeit in der Logik handelt.
\end{Regel}

\begin{RBsp}\begin{itemize}
	\item B\"ose: $\forall\varepsilon>0\exists n_0\in\N\forall n\geq n_0\colon \lvert a_n-a\rvert<\varepsilon$.

 	\item Gut: F\"ur jedes $\varepsilon>0$ existiert ein $n_0\in\N$, so dass $\lvert a_n-a\rvert<\varepsilon$ f\"ur alle $n\geq n_0$ gilt.

 	\item B\"ose: Daraus folgt $a=0\wedge b=1$.
 
  \item Gut: Daraus folgt $a=0$ und $b=1$.
\end{itemize}
\end{RBsp}

\begin{Regel}
Trennen Sie mathematische Symbole aus verschiedenen Formeln durch W\"orter.
\end{Regel}

\begin{RBsp}
 \begin{itemize}
  \item B\"ose: Es gilt f\"ur alle $n\in\N$ $\sum_{k=1}^nk=\frac{n(n+1)}{2}$.
  \item Gut: F\"ur alle $n\in\N$ gilt $\sum_{k=1}^nk=\frac{n(n+1)}{2}$.
  \item Gut: Es gilt f\"ur alle $n\in\N$ die Identit\"at $\sum_{k=1}^nk=\frac{n(n+1)}{2}$.
  \end{itemize}
\end{RBsp}


\begin{Regel}
  Mathematik sollte lesbar sein.
\end{Regel}

\begin{RBsp}
\begin{itemize}
  \item B\"ose: Sei $k>0$ eine ganze Zahl.
  \item Gut: Sei $k$ eine positive ganze Zahl.
  \item Gut: Betrachte eine ganze Zahl mit $k>0$.
  \item B\"ose: Sei $f\colon \R^n\supseteq D\to\R^m$ differenzierbar.
  \item Gut: Sei $D$ in $\R^n$ offen und $f\colon D\to\R^m$ differenzierbar.
 \end{itemize}
\end{RBsp}

 \end{Regel}

  \item Gut: Jede differenziebare Funktion ist stetig.
 \end{itemize}

	\item Verweisen Sie niemals auf eine Gleichungsnummer oder einen Gleichungsnamen, bevor die Gleichung überhaupt im Text auftaucht.

\begin{Regel}Lassen Sie die blo"se Aneinanderreihung von Gleichungen und Formeln sein und f\"uhren Sie stattdessen die Konzepte durch fortlaufende Kommentare zusammen.  
\end{Regel}
\begin{RBsp}\begin{itemize}
	\item Böse: \begin{align*}X^2-2X+2&=0\\
(X-1)^2+1&=0\\
(X-1)^2 &=-1\\
X-1&=\pm i\\
X&=1\pm i\
 \end{align*}
 
 \item Gut: Wir wollen die komplexen Nullstellen des Polynoms $X^2-2X+2$ bestimmen. Durch quadratische Erg\"anzung f\"uhrt dies auf die Gleichung \[(X-1)^2+1=0,\]
oder, \"aquivalent dazu, auf \[(X-1)^2=-1.\]
Davon sind die L\"osungen bekannt, n\"amlich $X-1$ ist gleich $i$ oder $-i$. Damit ist $X=1+i$ oder $X=1-i$.
\end{itemize}
\end{RBsp}
 
\begin{Regel}

\end{Regel}
 
\begin{RBsp}Sei $R$ ein einfacher kommutativer Ring mit Eins und seien $x$ und $y$ in $R$. Dann gilt $x^2-y^2=(x+y)(x-y)$.
\end{RBsp}
\begin{Bemerkung}
Der aufmerksame Leser wird sich fragen, was eine Eins oder die Tatsache, dass $R$ einfach ist, mit $x^2-y^2=(x+y)(x-y)$ zu tun hat. Hier wurden \emph{irrelevante} Informationen zu einer trivialen Aussage hinzugef\"ugt und rufen nur Verwirrung hervor.
\end{Bemerkung}

\begin{RBsp}
Jede komplexe Zahl ist das Produkt einer positiven Zahl und einer Zahl von Betrag eins.
\end{RBsp}

\begin{Bemerkung}
Hier wurde einfach der Spezialfall $0$ nicht erw\"ahnt: Diese komplexe Zahl kann nicht so dargestellt werden! Dies f\"uhrt zu Misstrauen beim Leser, wenn es h\"aufig passiert.
\end{Bemerkung}

\subsection{Unterschiede zwischen Satz, Proposition, Lemma und Korollar}\label{subsection:satz}Die Begriffe Satz, Proposition, Lemma und Korollar sind Ihnen wohl bekannt und Sie wissen auch, wie Sie diese mit \LaTeX{} einbinden können. Doch inwiefern unterscheiden sich diese Begriffe?

\begin{center}
\begin{tabular}{|p{60pt}|p{320pt}|}
\hline
Begriff& Nutzung\\
\hline
Satz& Ein Satz ist ein Hauptergebnis, welches von unabhängigem Interesse ist.\\
\hline
Proposition& Nicht alles, was bewiesen wird, ist ein Satz. Für kleinere und nützliche Ergebnisse, die auch von unabhängigem Interesse sind, verwendet man zur Abgrenzung zum Satz auch den Begriff Proposition.\\
\hline
Lemma& Ein Lemma ist ein Ergebnis, das ein Hilfsmittel für einen Satz oder eine Proposition darstellt, aber weniger von eigenständigem Interesse ist. Ein Lemma beinhaltet meist eine eher technische Aussage.\\
\hline
Korollar& Ein Korollar stellt eine unmittelbare Folgerung aus einem Satz oder einer Proposition dar, welche keinen oder lediglich einen kurzen Beweis erfordert.\\
\hline
\end{tabular}
\end{center}

\subsection{Struktur der Abschlussarbeit}\label{subsection:struktur}
Bei der Struktur Ihrer Abschlussarbeit k\"onnen Sie sich an folgenden Punkten orientieren:

\begin{itemize}
 \item Die Abschlussarbeit enth\"alt eine Einleitung inklusive Danksagung wie in dieser Vorlage erl\"autert.

 \item Im ersten Kapitel sollten Sie die Arbeit mathematisch vorbereiten, indem Sie zentrale Begriffe aus weiterf\"uhrenden Vorlesungen wiederholen und deren Notation fixieren sowie alle n\"otigen Begriffe und S\"atze zu \"uber Vorlesungen hinausgehenden Themen zusammenstellen. Es ist sinnvoll, mit dem Betreuer abzusprechen, wie dies genau auszusehen hat.

 \item Sie sollten die Arbeit inhaltlich mit einem resumierenden Kapitel abschlie\ss en, in welchem Sie auch auf offene und weiterf\"uhrende Fragen eingehen sollten.

 \item M\"ussen Sie in Ihrer Abschlussarbeit Rechnungen oder technische Beweise \"uber mehrere Seiten durchf\"uhren, so ist es f\"ur den Lesefluss f\"orderlich, wenn Sie diese in einen Anhang auslagern und darauf im Text verweisen.

 \item Ein Inhalts-, Literatur- und Stichwortverzeichnis wird erwartet.
\end{itemize}
